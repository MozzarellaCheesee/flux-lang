## Задача

Кодогенератор обходит AST и генерирует валидный C++20-код. Выходной файл можно скомпилировать напрямую любым C++ компилятором.

## Структура

```cpp
// include/flux/codegen/codegen.h
class CodeGen {
public:
    std::string generate(ProgramNode& program);
private:
    std::ostringstream out_;     // накапливаем C++ код
    int indent_ = 0;             // текущий уровень отступа

    void emit(const std::string& s);
    void emitln(const std::string& s);  // emit + перевод строки + отступ
    void indent();    // indent_++
    void dedent();    // indent_--
    // ...
};
```

Весь код накапливается в `std::ostringstream out_`. В конце `generate()` возвращает `out_.str()`. Это эффективнее, чем конкатенация строк через `+`.

## flux_runtime

Первое, что пишется в выходной файл — встроенный C++ runtime (~250 строк):

```cpp
std::string CodeGen::generate(ProgramNode& program) {
    out_ << R"(
// ============================================================
//  flux_runtime — автоматически вставляется компилятором Flux
// ============================================================
#include <cstdint>
#include <cmath>
#include <string>
#include <vector>
#include <iostream>
// ...

// Result<T, E>
template<typename T, typename E>
struct Result { ... };

// println, print, eprintln, eprint
template<typename... Args>
void println(Args&&... args) { (std::cout << ... << args); std::cout << '\n'; }

// io runtime
inline std::string read_line() { ... }

// math runtime
inline double sqrt(double x) { return std::sqrt(x); }
// ...

// string runtime
inline int32_t str_len(const std::string& s) { return (int32_t)s.size(); }
// ...
)";
    gen(program);
    return out_.str();
}
```

**Почему методы в c++, а не `#include <flux_runtime.h>`?** Пользователь получает только один файл — `fluxc.exe`. Нет никакой директории с заголовками. Код встроен в бинарник компилятора как строка C++ raw literal (`R"(...)"`).

## Маппинг типов

```cpp
std::string CodeGen::map_type_str(const std::string& t) {
    if (t == "bool")    return "bool";
    if (t == "int8")    return "int8_t";
    if (t == "int32")   return "int32_t";
    if (t == "int64")   return "int64_t";
    if (t == "float")   return "float";
    if (t == "double")  return "double";
    if (t == "str")     return "std::string_view";
    if (t == "&str")    return "std::string_view";
    if (t == "string")  return "std::string";
    if (t == "()")      return "void";
    return t;  // пользовательский тип — оставляем как есть
}

std::string CodeGen::map_type(TypeNode* t) {
    if (!t) return "void";
    if (auto* pt = dynamic_cast<PrimitiveType*>(t))
        return map_type_str(pt->name);
    if (auto* rt = dynamic_cast<RefType*>(t))
        return map_type(rt->inner.get()) + "&";
    if (auto* at = dynamic_cast<ArrayType*>(t))
        return "std::vector<" + map_type(at->element.get()) + ">";
    if (auto* gt = dynamic_cast<GenericType*>(t)) {
        if (gt->name == "Option")
            return "std::optional<" + map_type(gt->args[0].get()) + ">";
        // Result<T,E> и пользовательские дженерики
        std::string s = gt->name + "<";
        for (size_t i = 0; i < gt->args.size(); i++) {
            if (i) s += ", ";
            s += map_type(gt->args[i].get());
        }
        return s + ">";
    }
    if (dynamic_cast<UnitType*>(t)) return "void";
    return "auto";
}
```

### Решения по маппингу

| Flux | C++ | Обоснование                                  |
|------|-----|-------------|
| `string` | `std::string` | Владеющая строка — интуитивнее `std::string` |
| `&str` | `std::string_view` | Невладеющий слайс — аналог Rust `&str`       |
| `T[]` | `std::vector<T>` | Динамический массив — наиболее универсален   |
| `Option<T>` | `std::optional<T>` | Прямое соответствие в C++17                  |
| `Result<T,E>` | `Result<T,E>` | Собственный template в                       |
| `()` | `void` | Стандартное соглашение                       |

## Генерация объявлений

### Функции

```cpp
void CodeGen::gen(FuncDecl& node, const std::string& owner) {
    if (node.is_extern) return;  // extern fnc: реализация

    std::string ret = map_type(node.return_type.get());

    // Определяем имя с учётом владельца
    std::string fname;
    if (inside_class)  fname = sanitize_name(node.name);
    else if (is_method) fname = owner + "::" + sanitize_name(node.name);
    else               fname = sanitize_name(node.name);

    // Параметры (self пропускаем)
    std::string params;
    for (auto& param : node.params) {
        if (param->name == "self") continue;
        params += map_type(param->type.get()) + " " + param->name + ", ";
    }

    // static для методов без self
    std::string prefix = (is_method && !has_self) ? "static " : "";

    emitln(prefix + ret + " " + fname + "(" + params + ") {");
    indent();
    gen(*node.body);
    dedent();
    emitln("}");
}
```

### sanitize_name — резервированные имена C++

Некоторые идентификаторы Flux могут совпасть с ключевыми словами C++:

```cpp
std::string CodeGen::sanitize_name(const std::string& name) {
    static const std::unordered_set<std::string> reserved = {
        "new", "delete", "class", "template", "operator",
        "namespace", "try", "catch", "throw", "this"
    };
    if (reserved.count(name)) return "flux_" + name;
    return name;
}
```

Функция `new()` в Flux → `flux_new()` в C++.

### Структуры

```cpp
void CodeGen::gen(StructDecl& node) {
    emitln("struct " + node.name + " {");
    indent();
    for (auto& field : node.fields)
        emitln(map_type(field->type.get()) + " " + field->name + ";");
    dedent();
    emitln("};");
}
```

### Классы (struct + методы вместе)

```cpp
void CodeGen::gen(ClassDecl& node) {
    emitln("struct " + node.name + " {");
    indent();
    for (auto& field : node.fields)
        emitln(map_type(field->type.get()) + " " + field->name + ";");
    for (auto& method : node.methods)
        gen(*method, node.name);  // методы внутри struct
    dedent();
    emitln("};");
}
```

**Почему `class` → `struct` в C++?** В C++ `struct` и `class` различаются только доступностью по умолчанию. Для Flux, где доступ управляется явно через `pub`, `struct` с явными полями — более прозрачный выбор.

### impl-методы

```cpp
void CodeGen::gen(ProgramNode& node) {
    // Форвард-декларации
    for (auto& decl : node.decls) {
        if (auto* s = dynamic_cast<StructDecl*>(decl.get()))
            emitln("struct " + s->name + ";");
    }

    // Собираем impl-методы
    std::unordered_map<std::string, std::vector<FuncDecl*>> impl_methods;
    for (auto& decl : node.decls)
        if (auto* i = dynamic_cast<ImplDecl*>(decl.get()))
            for (auto& m : i->methods)
                impl_methods[i->target].push_back(m.get());

    // Struct + его методы вместе
    for (auto& decl : node.decls) {
        if (auto* s = dynamic_cast<StructDecl*>(decl.get())) {
            emitln("struct " + s->name + " {");
            indent();
            for (auto& field : s->fields) emitln(...);
            for (auto* m : impl_methods[s->name]) gen(*m, s->name);
            dedent();
            emitln("};");
        }
    }
}
```

Методы из `impl` вставляются прямо внутрь `struct`. Это C++-идиоматично и позволяет использовать `this->field` без дополнительных объявлений.

## Генерация выражений

Центральный метод — `expr(ASTNode&)` — возвращает строку C++ кода для выражения:

```cpp
std::string CodeGen::expr(ASTNode& node) {
    if (auto* n = dynamic_cast<IntLiteral*>(&node))     return std::to_string(n->value);
    if (auto* n = dynamic_cast<StrLiteral*>(&node))     return "std::string(" + n->value + ")";
    if (auto* n = dynamic_cast<BinaryExpr*>(&node))     return expr(*n);
    if (auto* n = dynamic_cast<CallExpr*>(&node))       return expr(*n);
    if (auto* n = dynamic_cast<MethodCallExpr*>(&node)) return expr(*n);
    if (auto* n = dynamic_cast<ImplicitCastExpr*>(&n))
        return "static_cast<" + map_type(n->target_type.get()) + ">(" + expr(*n->inner) + ")";
    // ...
}
```

**Почему dynamic_cast вместо virtual?** Кодогенератор — отдельный от Visitor класс. Использование `dynamic_cast` позволяет возвращать `std::string` из каждого метода (Visitor не может — у `visit` нет возвращаемого типа). Альтернатива — двойная диспетчеризация, но это сложнее для учебного проекта.

### Строковые литералы → `std::string(...)`

```cpp
std::string CodeGen::expr(StrLiteral& n) {
    return "std::string(" + n.value + ")";
}
```

Строковый литерал `"hello"` в Flux превращается в `std::string("hello")`, а не в `"hello"`. Это гарантирует, что тип будет `std::string`, а не `const char*`.

### Вызовы Ok/Err

```cpp
std::string CodeGen::expr(CallExpr& n) {
    if (n.callee == "Ok")  return "Result_Ok("  + expr(*n.args[0]) + ")";
    if (n.callee == "Err") return "Result_Err(" + expr(*n.args[0]) + ")";
    // обычный вызов
    return n.callee + "(" + args_str + ")";
}
```

### Вызовы методов

```cpp
std::string CodeGen::expr(MethodCallExpr& n) {
    // Встроенные методы
    if (n.method == "len")      return expr(*n.receiver) + ".size()";
    if (n.method == "push")     return expr(*n.receiver) + ".push_back(...)";
    if (n.method == "is_empty") return expr(*n.receiver) + ".empty()";

    // Статический вызов: TypeName.method() — если получатель с большой буквы
    if (auto* id = dynamic_cast<IdentExpr*>(n.receiver.get()))
        if (!id->name.empty() && std::isupper(id->name[0]))
            return id->name + "::" + sanitize_name(n.method) + "(" + args + ")";

    // self.method() → this->method()
    if (dynamic_cast<SelfExpr*>(n.receiver.get()))
        return "this->" + sanitize_name(n.method) + "(" + args + ")";

    return expr(*n.receiver) + "." + sanitize_name(n.method) + "(" + args + ")";
}
```

`match` транслируется в немедленно вызываемую лямбду `[&]() { ... }()`. Это позволяет использовать `match` как выражение (возвращающее значение) без дополнительной инфраструктуры.

### Инициализация структур → C++20 designated initializers

```cpp
std::string CodeGen::expr(StructInitExpr& n) {
    // Point { x: 1, y: 2 } → Point{.x=1, .y=2}
    std::string fields;
    for (auto& f : n.fields)
        fields += "." + f.name + "=" + expr(*f.value) + ", ";
    return n.type_name + "{" + fields + "}";
}
```

**Зачем C++20 designated initializers?** Они позволяют инициализировать поля по имени, а не по позиции. Это соответствует синтаксису Flux `Point { x: 1, y: 2 }` и устойчиво к изменению порядка полей.

## Форвард-декларации

В начале генерации для всех `struct` и `class` выводятся форвард-декларации:

```cpp
// Форвард-декларации предотвращают ошибки при взаимных ссылках
struct Point;
struct Circle;

struct Point { ... };
struct Circle {
    Point center;   // Point уже известен из форвард-декларации
    ...
};
```
