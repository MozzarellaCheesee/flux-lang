## Что такое Flux?

Flux — язык программирования, разработанный в целях понимания как устроен frontend языка. Цель проекта — изучить принципы построения компиляторов: лексический анализ, синтаксический разбор, семантическую проверку и генерацию кода.

Flux — **транспилирующийся** язык: компилятор `fluxc.exe` читает `.flx`-файл и генерирует C++20-код, который затем компилируется установленным в системе C++-компилятором (GCC, Clang или MSVC).

## Ключевые решения проекта

### Почему транспиляция в C++, а не генерация машинного кода?

Прямая генерация машинного кода (x86, ARM) требует глубокого знания ABI, соглашений о вызовах, распределения регистров и линковки — это отдельная дисциплина, выходящая далеко за рамки моего учебного проекта.

Транспиляция в C++ даёт несколько преимуществ:
- **Готовая оптимизация** — все оптимизации выполняет GCC/Clang
- **Кроссплатформенность** — тот же .flx работает на Windows, Linux, macOS
- **Отладка** — при ошибке выполнения можно посмотреть сгенерированный .cpp
- **Стандартная библиотека** — C++ STL (`std::vector`, `std::string`) доступна бесплатно

Недостаток: сообщения об ошибках C++ компилятора могут "просочиться" к пользователю при ошибках, которые семантический анализатор Flux не поймал.
### Один исполняемый файл

Пользователь получает только `fluxc.exe` — больше никаких дополнительных файлов. Стандартная библиотека и C++ runtime встроены прямо в бинарник компилятора как строки. При `#import <io>` препроцессор достаёт Flux-исходник модуля из памяти, не обращаясь к диску.

## Конвейер компиляции

```
source.flx
    │
    ▼
┌─────────┐
│  Lexer  │  Токенизация: текст → []Token
└────┬────┘
     │
     ▼
┌──────────────┐
│ Preprocessor │  Раскрывает #import (файловые и stdlib)
└──────┬───────┘
       │
       ▼
┌────────┐
│ Parser │  Tokens → AST (ProgramNode)
└───┬────┘
    │
    ▼
┌───────────────────┐
│ SemanticAnalyzer  │  Проверка типов, регистрация функций
└────────┬──────────┘
         │
         ▼
    ┌─────────┐
    │ CodeGen │  AST → C++ исходник
    └────┬────┘
         │
         ▼
┌────────────────┐
│ C++ Compiler   │  g++ / clang++ / cl (внешний)
│ (GCC/Clang/MSVC│
└────────┬───────┘
         │
         ▼
      program.exe
```

Каждый этап изолирован и общается только с соседними через чётко определённые структуры данных:
- Lexer → Preprocessor: `std::vector<Token>`
- Preprocessor → Parser: `std::vector<Token>`
- Parser → Sema: `std::unique_ptr<ProgramNode>`
- Sema → CodeGen: тот же `ProgramNode` (изменений нет)
- CodeGen → C++ compiler: файл `.cpp`

## Технологии

| Компонент          | Технология                    |
| ------------------ | ----------------------------- |
| Язык реализации    | C++20                         |
| Система сборки     | CMake 3.28+                   |
| Управление памятью | `std::unique_ptr` (владение)  |
| Паттерн обхода AST | Visitor (заход в каждую ноду) |
| Парсинг выражений  | Pratt (binding power)         |
