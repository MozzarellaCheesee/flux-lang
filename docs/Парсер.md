## Задача парсера

Парсер превращает плоский поток токенов в **абстрактное синтаксическое дерево (AST)** — иерархическую структуру, отражающую грамматику программы.

```
[KW_FNC] [IDENT "add"] [LPAREN] [IDENT "a"] [COLON] [KW_INT32] ...
    →
FuncDecl {
    name: "add"
    params: [ParamDecl{name:"a", type:PrimitiveType{"int32"}},
             ParamDecl{name:"b", type:PrimitiveType{"int32"}}]
    return_type: PrimitiveType{"int32"}
    body: BlockStmt {
        stmts: [ReturnStmt{BinaryExpr{"+", IdentExpr{"a"}, IdentExpr{"b"}}}]
    }
}
```

## Метод разбора

Используется **рекурсивный спуск** для объявлений и операторов, **алгоритм Пратта** (Pratt parsing / top-down operator precedence) для выражений.

**Pratt-парсер (парсер Пратта)** — это ==эффективный и гибкий метод синтаксического анализа (parsing) выражений==, предложенный Воганом Праттом в 1973 году. Он основан на рекурсивном спуске, но вместо жестких правил грамматики связывает семантику (функции обработки) непосредственно с лексемами (токенами), что делает его идеальным для разбора выражений с операторами, приоритетами и ассоциативностью.
Это ключевая часть. Обычный рекурсивный спуск не справляется с приоритетами операторов без отдельной функции на каждый уровень приоритета. Pratt решает это одной функцией и таблицей `binding power`.
`Binding power` (BP) — сила притяжения оператора к операндам. Чем больше число, тем сильнее оператор "притягивает" соседние операнды.
Каждый инфиксный оператор имеет **два** числа:
- **Левый BP (`lbp`)** — насколько сильно оператор притягивает левый операнд
- **Правый BP (`rbp`)** — насколько сильно притягивает правый операнд

### Почему Pratt для выражений?

Классический рекурсивный спуск требует написать отдельный метод для каждого уровня приоритета (`parse_additive`, `parse_multiplicative`, `parse_unary`...). При 10+ уровнях приоритета это неудобно.

Алгоритм Пратта использует таблицу **binding power** (силы притяжения) для каждого оператора и одну универсальную функцию `parse_expr(min_bp)`:

```cpp
std::unique_ptr<ASTNode> Parser::parse_expr(int min_bp) {
    auto lhs = parse_prefix();   // унарные операторы, атомы

    while (true) {
        auto [left_bp, right_bp] = infix_bp(current().kind);
        if (left_bp <= min_bp) break;
        // берём оператор, рекурсивно читаем правую часть
        auto op = consume();
        auto rhs = parse_expr(right_bp);
        lhs = make_binary(op.lexeme, std::move(lhs), std::move(rhs));
    }
    return lhs;
}
```

Таблица binding power:
| Операторы | left_bp | right_bp |
|-----------|---------|----------|
| `\|\|` | 10 | 11 |
| `&&` | 20 | 21 |
| `==`, `!=` | 30 | 31 |
| `<`, `>`, `<=`, `>=` | 40 | 41 |
| `+`, `-` | 50 | 51 |
| `*`, `/`, `%` | 60 | 61 |

Чем выше значение — тем сильнее связывает оператор.

## Структура парсера

```cpp
class Parser {
    // Вспомогательные методы
    Token consume();                    // взять текущий токен и продвинуться
    Token expect(TokenKind, const char* msg); // ожидаем конкретный токен
    bool  match(TokenKind);             // consume если совпадает
    bool  check(TokenKind) const;       // проверить без потребления

    // Верхний уровень
    std::unique_ptr<ProgramNode> parse_program();
    std::unique_ptr<ASTNode>     parse_top_level();

    // Объявления
    std::unique_ptr<FuncDecl>   parse_func_decl(bool is_pub, bool is_extern = false);
    std::unique_ptr<StructDecl> parse_struct_decl(bool is_pub);
    std::unique_ptr<ImplDecl>   parse_impl_decl();
    std::unique_ptr<ClassDecl>  parse_class_decl(bool is_pub);
    std::unique_ptr<FieldDecl>  parse_field_decl();

    // Операторы
    std::unique_ptr<ASTNode>    parse_stmt();
    std::unique_ptr<BlockStmt>  parse_block();
    std::unique_ptr<VarDecl>    parse_let_stmt();
    // ...

    // Выражения
    std::unique_ptr<ASTNode>    parse_expr(int min_bp = 0);
    std::unique_ptr<ASTNode>    parse_prefix();
    std::unique_ptr<ASTNode>    parse_postfix(std::unique_ptr<ASTNode> lhs);
    std::unique_ptr<ASTNode>    parse_atom();
};
```

## Разбор верхнего уровня

`parse_top_level()` определяет, что за конструкция следует, по первому токену:

```cpp
std::unique_ptr<ASTNode> Parser::parse_top_level() {
    bool is_pub = match(KW_PUB);     // необязательный pub

    bool is_extern = false;
    if (match(KW_EXTERN)) is_extern = true;  // необязательный extern

    if (check(KW_FNC))    return parse_func_decl(is_pub, is_extern);
    if (check(KW_STRUCT)) return parse_struct_decl(is_pub);
    if (check(KW_CLASS))  return parse_class_decl(is_pub);
    if (check(KW_IMPL))   return parse_impl_decl();
    if (check(PP_IMPORT)) { consume(); expect(LIT_STRING); return nullptr; } // уже обработан
    if (check(PP_IMPORT_SYS)) { consume(); return nullptr; }

    // Неожиданный токен — ошибка + recovery
    diag_.emit(Error, current().loc, "Expected top-level declaration");
    consume(); // recovery: съедаем и идём дальше
    return nullptr;
}
```

## Разбор функции

```cpp
// fnc name(params) -> ReturnType { body }
// extern fnc name(params) -> ReturnType;
std::unique_ptr<FuncDecl> Parser::parse_func_decl(bool is_pub, bool is_extern) {
    expect(KW_FNC, "fnc");
    auto name = std::string(expect(IDENT, "function name").lexeme);
    expect(LPAREN, "(");

    // self как первый параметр — только в методах
    bool has_self = false;
    if (check(KW_SELF)) { consume(); has_self = true; match(COMMA); }

    // остальные параметры: name: Type, name: Type, ...
    std::vector<std::unique_ptr<ParamDecl>> params;
    while (!check(RPAREN)) {
        auto pname = std::string(expect(IDENT, "parameter name").lexeme);
        expect(COLON, ":");
        auto ptype = parse_type();
        params.push_back(std::make_unique<ParamDecl>(pname, std::move(ptype)));
        if (!match(COMMA)) break;
    }
    expect(RPAREN, ")");
    expect(ARROW, "->");
    auto ret = parse_type();

    std::unique_ptr<BlockStmt> body;
    if (is_extern) {
        match(SEMICOLON); // ; необязательна
    } else {
        body = parse_block();
    }
    return std::make_unique<FuncDecl>(is_pub, is_extern, has_self, name,
                                      std::move(params), std::move(ret), std::move(body));
}
```

## Разбор типов

```cpp
std::unique_ptr<TypeNode> Parser::parse_type() {
    // &T — ссылочный тип
    if (match(AMP)) return std::make_unique<RefType>(parse_type());

    // [] — слайс
    if (match(LBRACKET)) { expect(RBRACKET); return std::make_unique<SliceType>(parse_type()); }

    // ( ) — unit-тип
    if (match(LPAREN)) { expect(RPAREN); return std::make_unique<UnitType>(); }

    // Имя типа — примитив или пользовательский
    auto name = token_to_type_name(current());
    consume();

    // Generic: Name<T, U>
    if (match(LT)) {
        auto gt = std::make_unique<GenericType>(name);
        while (!check(GT)) {
            gt->args.push_back(parse_type());
            if (!match(COMMA)) break;
        }
        expect(GT);
        return gt;
    }

    // T[] — массив (после имени)
    if (match(LBRACKET)) {
        expect(RBRACKET);
        return std::make_unique<ArrayType>(
            std::make_unique<PrimitiveType>(name), nullptr);
    }

    return std::make_unique<PrimitiveType>(name);
}
```

## Разбор выражений — атомы

Атомы — наименьшие неделимые выражения:

```cpp
std::unique_ptr<ASTNode> Parser::parse_atom() {
    switch (current().kind) {
        case LIT_INT:    return make_unique<IntLiteral>(stol(consume().lexeme));
        case LIT_FLOAT:  return make_unique<FloatLiteral>(stod(consume().lexeme));
        case LIT_STRING: return make_unique<StrLiteral>(consume().lexeme);
        case KW_TRUE:    consume(); return make_unique<BoolLiteral>(true);
        case KW_FALSE:   consume(); return make_unique<BoolLiteral>(false);
        case KW_SELF:    consume(); return make_unique<SelfExpr>();
        case LBRACKET:   return parse_array_literal();
        case LPAREN:     // группировка (expr)
            consume();
            auto e = parse_expr();
            expect(RPAREN);
            return e;
        case IDENT: {
            auto name = consume().lexeme;
            if (check(LPAREN)) return parse_call(name);  // вызов функции
            if (check(LBRACE) && allow_struct_init_)     // инициализация структуры
                return parse_struct_init(name);
            return make_unique<IdentExpr>(name);
        }
        case KW_MATCH: return parse_match_expr();
    }
}
```

## Постфиксные операции

После атома обрабатываются постфиксные операции:

```cpp
std::unique_ptr<ASTNode> Parser::parse_postfix(std::unique_ptr<ASTNode> lhs) {
    while (true) {
        if (match(DOT)) {
            auto field = expect(IDENT).lexeme;
            if (check(LPAREN)) {
                // method call: expr.method(args)
                auto call = make_unique<MethodCallExpr>(std::move(lhs), field);
                parse_call_args(call->args);
                lhs = std::move(call);
            } else {
                lhs = make_unique<FieldAccessExpr>(std::move(lhs), field);
            }
        } else if (match(LBRACKET)) {
            // индексация: expr[idx]
            auto idx = parse_expr();
            expect(RBRACKET);
            lhs = make_unique<IndexExpr>(std::move(lhs), std::move(idx));
        } else if (check(PLUS_PLUS)) {
            consume();
            lhs = make_unique<UnaryExpr>(PostInc, std::move(lhs));
        } else {
            break;
        }
    }
    return lhs;
}
```

## Восстановление после ошибок

При неожиданном токене парсер:
1. Записывает ошибку через `diag_.emit()`
2. **Не продвигает курсор** — вышестоящий код может попробовать другой разбор
3. Или потребляет один токен и продолжает (recovery)

Это позволяет показывать несколько ошибок за один запуск компилятора вместо остановки на первой.
