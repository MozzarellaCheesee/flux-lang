## Задача препроцессора

Препроцессор работает **после** лексера, но **до** парсера. Его единственная задача — раскрыть директивы `#import`, подставив вместо них содержимое указанного модуля.

На выходе парсер получает единый плоский поток токенов, будто весь код был написан в одном файле.

## Почему препроцессор, а не модульная система?

В полноценных языках (Rust, Go) существует модульная система на уровне семантики. В Flux для упрощения реализована **текстовая подстановка** — аналог `#include` в C/C++.

## API

```cpp
// include/flux/preprocessor/preprocessor.h
class Preprocessor {
public:
    Preprocessor(DiagEngine& diag, std::filesystem::path base_dir);

    std::vector<Token> process(
        const std::vector<Token>& tokens,
        const std::filesystem::path& current_file
    );
private:
    std::vector<Token> load_and_tokenize(const std::filesystem::path& path);

    DiagEngine& diag_;
    std::filesystem::path base_dir_;
    std::unordered_set<std::string> visited_; // защита от циклов
    std::vector<std::string> source_store_;   // хранит строки файлов
};
```

## Алгоритм

```
для каждого токена:
    если это PP_IMPORT:
        берём следующий токен (LIT_STRING с путём)
        вычисляем полный путь: current_file.parent / path
        нормализуем через weakly_canonical
        если уже в visited_ → предупреждение, пропускаем
        добавляем в visited_
        читаем файл → строка
        лексируем строку → токены
        рекурсивно process(токены, import_path)
        добавляем результат в output (без EOF)

    если это PP_IMPORT_SYS:
        module_name = tok.lexeme   ("io", "math", "string")
        key = "__stdlib_" + module_name
        если key в visited_ → пропускаем
        добавляем key в visited_
        ищем в STDLIB_MODULES[module_name]
        если не найден → ошибка
        лексируем встроенный исходник
        рекурсивно process(токены, current_file)
        добавляем результат в output

    иначе → копируем токен в output
```

## Защита от циклических импортов

Рассмотрим пример:
```
a.flx:
    #import "b.flx"

b.flx:
    #import "a.flx"   ← цикл!
```

Множество `visited_` хранит канонические пути всех уже обработанных файлов. При повторной встрече тот же путь — пропуск с предупреждением:

```cpp
std::string canonical = import_path.string();
if (visited_.count(canonical)) {
    diag_.emit(DiagLevel::Warning, tok.loc, "Circular import ignored: " + canonical);
    continue;
}
visited_.insert(canonical);
```

Для stdlib используется специальный префикс `__stdlib_`, который никогда не совпадёт с путём реального файла.

## source_store_

Лексер принимает `std::string_view` — невладеющий указатель на строку. Токены хранят `string_view` лексемы из этой строки. Если строка будет уничтожена — все `string_view` станут висящими указателями.

`source_store_` — вектор строк, которому принадлежат все исходники импортированных файлов. Пока `Preprocessor` жив, все строки валидны.

```cpp
source_store_.push_back(std::move(source));    // строка переходит во владение
const std::string& stored = source_store_.back();
Lexer lexer(stored, path.string().c_str(), diag_);  // lexer берёт view
```

## Stdlib-модули

Встроенные модули хранятся в `include/flux/stdlib/stdlib_modules.h`:

```cpp
static const std::unordered_map<std::string, std::string> STDLIB_MODULES = {
    {"io",     "extern fnc read_line() -> string;\n..."},
    {"math",   "extern fnc sqrt(x: double) -> double;\n..."},
    {"string", "extern fnc str_len(s: string) -> int32;\n..."},
};
```

При `#import <math>` препроцессор берёт строку из этой таблицы, лексирует и рекурсивно обрабатывает. Токены `extern fnc` попадают в общий поток — парсер разбирает их как обычные объявления.

## Пример трассировки

```flux
// main.flx
#import <math>
fnc main() -> int32 {
    let x = sqrt(16.0);
    return 0;
}
```

Поток токенов после препроцессора:
```
[KW_EXTERN] [KW_FNC] [IDENT "sqrt"] [LPAREN] [IDENT "x"] [COLON] [KW_DOUBLE]
[RPAREN] [ARROW] [KW_DOUBLE] [SEMICOLON]
... (остальные extern fnc из math)
[KW_FNC] [IDENT "main"] [LPAREN] [RPAREN] [ARROW] [KW_INT32] [LBRACE]
[KW_LET] [IDENT "x"] [EQ] [IDENT "sqrt"] [LPAREN] [LIT_FLOAT "16.0"] [RPAREN] [SEMICOLON]
[KW_RETURN] [LIT_INT "0"] [SEMICOLON]
[RBRACE] [EOF]
```
