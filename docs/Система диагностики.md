## Структуры

```cpp
// include/flux/common/diagnostic.h

enum class DiagLevel { Error, Warning, Note };

struct Diagnostic {
    DiagLevel      level;
    SourceLocation loc;      // файл, строка, столбец
    std::string    message;
};

class DiagEngine {
public:
    void   emit(DiagLevel level, SourceLocation loc, std::string message);
    bool   has_errors() const;
    size_t error_count() const;
    void   print_all() const;
private:
    std::vector<Diagnostic> diags_;
    size_t error_count_ = 0;
};
```

## Принцип работы

`DiagEngine` накапливает диагностику — не бросает исключения и не прерывает компиляцию. Это позволяет **показать несколько ошибок за раз**:

```
main.flx:5:12: error: Undefined function 'foo'
main.flx:8:3: error: Cannot assign '&str' to variable of type 'int32'
main.flx:12:1: warning: Variable 'x' shadows previous declaration
```

Вместо того чтобы остановиться на первой ошибке, компилятор продолжает анализ и выводит все найденные проблемы.

## Уровни диагностики

| Уровень | Останавливает компиляцию? | Пример |
|---------|--------------------------|--------|
| `Error` | Да (после текущего этапа) | Неизвестная переменная |
| `Warning` | Нет | Shadowing переменной |
| `Note` | Нет | Дополнительный контекст |

## Проверка после каждого этапа

```cpp
// main.cpp
flux::Lexer lexer(source, ...);
auto tokens = lexer.tokenize();
if (diag.has_errors()) { diag.print_all(); return 1; }

flux::Preprocessor preprocessor(diag, ...);
auto processed = preprocessor.process(tokens, ...);
if (diag.has_errors()) { diag.print_all(); return 2; }

flux::Parser parser(std::move(processed), diag);
auto program = parser.parse_program();
if (diag.has_errors()) { diag.print_all(); return 3; }

flux::SemanticAnalyzer sema(diag);
sema.analyze(*program);
if (diag.has_errors()) { diag.print_all(); return 5; }
```

Каждый этап может добавить ошибки. После этапа проверяется флаг — если есть ошибки, выводим всё накопленное и выходим.

## SourceLocation

```cpp
// include/flux/common/source_location.h
struct SourceLocation {
    std::string filepath;
    uint32_t    line;
    uint32_t    col;
};
```

Пустой `SourceLocation{}` используется для общих ошибок без привязки к конкретной строке (например, "main function not found").

## Пример вывода

```
Examples/main.flx:7:5: error: Undefined function 'baz'
Examples/main.flx:7:5: error: No matching overload for 'add' with args (string). Available overloads:
  add(int32, int32) -> int32
Examples/main.flx:12:14: warning: Variable 'result' shadows previous declaration
```

Формат: `файл:строка:столбец: уровень: сообщение`
