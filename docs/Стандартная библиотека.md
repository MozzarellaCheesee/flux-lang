## Концепция

Стандартная библиотека Flux состоит из двух слоёв:

```
┌─────────────────────────────────────────────────────┐
│                    fluxc.exe                        │
│                                                     │
│  ┌──────────────────────────────────────────────┐   │
│  │  flux_runtime (C++)                          │   │
│  │  Встраивается в КАЖДУЮ программу             │   │
│  │  • println, print, eprintln, eprint          │   │
│  │  • Result<T,E>                               │   │
│  │  • io: read_line, read_int...                │   │
│  │  • math: sqrt, floor, sin...                 │   │
│  │  • string: str_len, str_to_upper...          │   │
│  └──────────────────────────────────────────────┘   │
│                                                     │
│  ┌──────────────────────────────────────────────┐   │
│  │  Flux-модули (extern fnc декларации)         │   │
│  │  Хранятся как строки внутри exe              │   │
│  │  #import <io>     → extern fnc read_line...  │   │
│  │  #import <math>   → extern fnc sqrt...       │   │
│  │  #import <string> → extern fnc str_len...    │   │
│  └──────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### Почему два слоя?

**C++** необходим потому, что некоторые функции невозможно реализовать на самом Flux (нет доступа к C-функциям), — поэтому их реализации написаны на C++ и буквально включены в каждый генерируемый `.cpp` файл.

**Flux-модули** (`extern fnc` объявления) нужны для того, чтобы **семантический анализатор** знал о существовании этих функций — их типах параметров и возвращаемых типах. Без этих объявлений анализатор выдал бы ошибку "undefined function 'sqrt'".

Когда пользователь пишет `#import <math>`, препроцессор вставляет в поток токенов набор `extern fnc` объявлений. Парсер разбирает их как обычные `FuncDecl` с флагом `is_extern = true`. Кодогенератор для таких объявлений ничего не генерирует — реализация уже есть.

## Всегда доступные функции (без #import)

Эти функции работают без импортов:

### Вывод

```flux
println(a, b, c, ...)     // вывод в stdout + перевод строки
print(a, b, c, ...)       // вывод в stdout
eprintln(a, b, c, ...)    // вывод в stderr + перевод строки
eprint(a, b, c, ...)      // вывод в stderr
```

`println` принимает **произвольное количество аргументов любых типов** благодаря variadic templates C++:
```cpp
template<typename... Args>
void println(Args&&... args) { (std::cout << ... << args); std::cout << '\n'; }
```

### Result<T, E>

```flux
let r: Result<int32, string> = Ok(42);
let e: Result<int32, string> = Err("oops");

r.is_ok()       // true
r.is_err()      // false
r.unwrap()      // 42
e.unwrap_err()  // "oops"
```

## #import \<io\>

```flux
#import <io>
```

| Функция | Тип возврата | Описание |
|---------|-------------|----------|
| `read_line()` | `string` | Читает строку со stdin (до `\n`) |
| `read_int()` | `int32` | Читает целое число со stdin |
| `read_uint()` | `uint32` | Читает беззнаковое целое |
| `read_float()` | `float` | Читает число с плавающей точкой |
| `read_double()` | `double` | Читает число двойной точности |

### Пример

```flux
#import <io>

fnc main() -> int32 {
    println("Введите число:");
    let n: int32 = read_int();
    println("Вы ввели: ", n);
    return 0;
}
```

### Реализация на C++

```cpp
inline std::string read_line() {
    std::string s;
    std::getline(std::cin, s);
    return s;
}
inline int32_t read_int() {
    int32_t n = 0;
    std::cin >> n;
    std::cin.ignore();  // убираем \n после числа
    return n;
}
```

## #import \<math\>

```flux
#import <math>
```

### Основные математические функции

| Функция | Описание |
|---------|----------|
| `sqrt(x: double) -> double` | Квадратный корень |
| `cbrt(x: double) -> double` | Кубический корень |
| `pow(base: double, exp: double) -> double` | Возведение в степень |
| `abs_i32(x: int32) -> int32` | Абсолютное значение int32 |
| `abs_i64(x: int64) -> int64` | Абсолютное значение int64 |
| `abs_f64(x: double) -> double` | Абсолютное значение double |
| `floor(x: double) -> double` | Округление вниз |
| `ceil(x: double) -> double` | Округление вверх |
| `round(x: double) -> double` | Округление к ближайшему |
| `trunc(x: double) -> double` | Усечение дробной части |

### Логарифмы и экспонента

| Функция | Описание |
|---------|----------|
| `log(x: double) -> double` | Натуральный логарифм |
| `log2(x: double) -> double` | Логарифм по основанию 2 |
| `log10(x: double) -> double` | Логарифм по основанию 10 |
| `exp(x: double) -> double` | e^x |

### Тригонометрия

| Функция | Описание |
|---------|----------|
| `sin(x: double) -> double` | Синус (радианы) |
| `cos(x: double) -> double` | Косинус (радианы) |
| `tan(x: double) -> double` | Тангенс (радианы) |
| `asin(x: double) -> double` | Арксинус |
| `acos(x: double) -> double` | Арккосинус |
| `atan(x: double) -> double` | Арктангенс |
| `atan2(y: double, x: double) -> double` | Двухаргументный арктангенс |

### Min / Max / Clamp

| Функция | Описание |
|---------|----------|
| `min_i32(a: int32, b: int32) -> int32` | Минимум двух int32 |
| `max_i32(a: int32, b: int32) -> int32` | Максимум двух int32 |
| `min_i64(a: int64, b: int64) -> int64` | Минимум двух int64 |
| `max_i64(a: int64, b: int64) -> int64` | Максимум двух int64 |
| `min_f64(a: double, b: double) -> double` | Минимум двух double |
| `max_f64(a: double, b: double) -> double` | Максимум двух double |
| `clamp_i32(v: int32, lo: int32, hi: int32) -> int32` | Ограничение диапазона |
| `clamp_f64(v: double, lo: double, hi: double) -> double` | Ограничение диапазона |

### Пример

```flux
#import <math>

fnc main() -> int32 {
    let side: double = 3.0;
    let hyp: double = sqrt(side * side + side * side);
    println("Гипотенуза: ", hyp);

    let angle_rad: double = atan2(3.0, 4.0);
    println("Угол: ", angle_rad);

    let clamped: int32 = clamp_i32(150, 0, 100);
    println("Clamped: ", clamped);  // 100
    return 0;
}
```

## #import \<string\>

```flux
#import <string>
```

### Информация о строке

| Функция | Описание |
|---------|----------|
| `str_len(s: string) -> int32` | Длина строки в байтах |
| `str_is_empty(s: string) -> bool` | Пустая ли строка |
| `str_char_at(s: string, i: int32) -> int32` | Код символа по индексу (–1 если вне границ) |

### Поиск и проверка

| Функция | Описание |
|---------|----------|
| `str_contains(s: string, sub: string) -> bool` | Содержит ли подстроку |
| `str_starts_with(s: string, prefix: string) -> bool` | Начинается ли с префикса |
| `str_ends_with(s: string, suffix: string) -> bool` | Заканчивается ли суффиксом |

### Преобразования

| Функция                                                      | Описание                      |
| ------------------------------------------------------------ | ----------------------------- |
| `str_to_upper(s: string) -> string`                          | В верхний регистр             |
| `str_to_lower(s: string) -> string`                          | В нижний регистр              |
| `str_trim(s: string) -> string`                              | Убрать пробелы с обоих концов |
| `str_trim_start(s: string) -> string`                        | Убрать пробелы слева          |
| `str_trim_end(s: string) -> string`                          | Убрать пробелы справа         |
| `str_replace(s: string, from: string, to: string) -> string` | Заменить все вхождения        |
| `str_repeat(s: string, n: int32) -> string`                  | Повторить строку n раз        |
| `str_concat(a: string, b: string) -> string`                 | Конкатенация                  |

### Конвертация

| Функция | Описание |
|---------|----------|
| `to_string_i32(n: int32) -> string` | Число → строка |
| `to_string_i64(n: int64) -> string` | Число → строка |
| `to_string_f64(n: double) -> string` | Число → строка |
| `to_string_bool(b: bool) -> string` | bool → "true"/"false" |
| `parse_int(s: string) -> int32` | Строка → int32 |
| `parse_float(s: string) -> double` | Строка → double |

### Пример

```flux
#import <string>

fnc main() -> int32 {
    let s: string = "  Hello, World!  ";
    let trimmed: string = str_trim(s);
    let upper: string   = str_to_upper(trimmed);
    let replaced: string = str_replace(upper, "WORLD", "FLUX");

    println(replaced);  // "HELLO, FLUX!"

    let n: string = to_string_i32(42);
    println("Number as string: ", n);

    let parsed: int32 = parse_int("123");
    println("Parsed: ", parsed);  // 123

    return 0;
}
```

## Как добавить новый stdlib-модуль

1. Написать реализацию функции в C++ (`src/codegen/codegen.cpp`, в блоке `R"(...)"`):
   ```cpp
   inline int32_t my_func(int32_t x) { return x * 2; }
   ```

2. Добавить `extern fnc` объявление в `include/flux/stdlib/stdlib_modules.h`:
   ```cpp
   {"mymodule", R"flux(
   extern fnc my_func(x: int32) -> int32;
   )flux"},
   ```

3. Пользователь пишет:
   ```flux
   #import <mymodule>
   fnc main() -> int32 {
       println(my_func(21));  // 42
       return 0;
   }
   ```
